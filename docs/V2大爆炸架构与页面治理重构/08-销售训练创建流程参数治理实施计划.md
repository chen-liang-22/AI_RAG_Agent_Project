# 销售训练创建流程参数治理 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修正销售训练“创建训练”流程，确保第一步只创建训练名称，学员画像、客户画像和场景描述必须由用户确认后再保存并参与 AI 客户生成。

**Architecture:** 后端把训练方案创建拆成“空方案创建”和“画像场景快照保存”两个阶段；前端把默认表单值降级为草稿，不再把它们当作已选择参数提交。生成 AI 客户前统一通过 `PUT /api/v2/training/plans/{plan_id}` 保存当前确认过的画像和场景，再调用补充问答和角色生成接口。

**Tech Stack:** FastAPI、Pydantic、SQLAlchemy、pytest、Vue 3、TypeScript、Element Plus、Vite。

---

## 1. Scope

本计划只处理销售训练创建流程参数治理，不扩大到训练资料上传、评分规则生成、聊天会话、考试模块或 V2 旧依赖删除。

### In Scope

- 创建训练时只提交 `plan_name`。
- 后端 `TrainingPlanCreateRequest` 支持名称单独创建。
- 空训练方案落库时不写入默认学员画像、默认客户画像、默认场景。
- 前端未确认画像前显示“未配置”，不能直接生成 AI 客户。
- 生成 AI 客户前先保存用户确认的画像、客户字段和场景快照。
- 补测试覆盖空方案创建和首次补齐画像状态。

### Out Of Scope

- 不重构销售训练大页面组件拆分。
- 不新增客户画像 code/label 双结构，后续可单独规划。
- 不修改训练资料 MD5 去重逻辑。
- 不删除旧训练服务或旧路由。

async function saveCurrentPlanProfileSnapshot() {

    openCustomerProfileDialog()
    return false
  }
  if (!scenarioDescription.value.trim()) {
    ElMessage.warning('请填写训练场景描述')
    openCustomerProfileDialog()
    return false
  }

  const detail = await updateTrainingPlan(activePlan.value.plan_id, {
    ...currentPlanPayload(),
    scenario_description: scenarioDescription.value.trim(),
    extra_details: extraDetails.value.trim(),
  })
  hydratePlanDetail(detail)
  traineeProfileConfirmed.value = true
  customerProfileConfirmed.value = true
  await refreshTrainingPlans()
  return true
}

| 验收点 | 标准 |
| --- | --- |
| 创建训练请求 | `POST /api/v2/training/plans` 只包含 `plan_name` |
| 空方案落库 | `trainee_json={}`、`selected_fields_json={}`、`scenario_description=""` |
| 默认画像 | 页面默认值只作为草稿，不作为已确认参数 |
| 学员画像 | 用户点击确定后才显示已配置 |
| 客户画像 | 用户点击确定后才显示已配置 |
| 生成 AI 客户 | 必须先保存方案快照，再生成补充问题和角色 |
| 状态语义 | 首次补齐画像仍为 `pending/pending/pending`，已有角色后修改输入才进入 `stale` |
| 后端测试 | `pytest -q` 全量通过 |
| 前端构建 | `npm run build` 通过 |
