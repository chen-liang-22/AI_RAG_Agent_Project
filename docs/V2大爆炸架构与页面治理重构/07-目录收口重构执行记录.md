# 目录收口重构执行记录

## 1. 目标

本次收口解决“大爆炸重构后目录仍然过多、旧目录和 V2 目录并存”的问题。

核心目标：

- `api/` 只保留 FastAPI 启动入口和启动预热。
- `app_v2/` 承接业务路由、应用服务、领域模型和基础设施。
- `core/` 承接可复用底层能力，包括 Agent、模型工厂、RAG 和通用工具。
- 删除外层 `agent/`、`model/`、`rag/`、`utils/`、`domain/`、`infrastructure/`、`training/` 等并列目录。

## 3. 迁移明细

### 3.1 基础能力迁入 core

| 旧目录 | 新目录 | 说明 |
| --- | --- | --- |
| `agent/` | `core/agent/` | ReAct Agent 和工具函数 |
| `model/` | `core/model/` | LLM、Embedding 工厂 |
| `rag/` | `core/rag/` | RAG 检索、切分、精排、答案生成 |
| `utils/` | `core/utils/` | 配置、日志、路径、文件、MinIO、Redis、Qdrant 配置工具 |

### 3.2 业务领域和基础设施迁入 app_v2

| 旧目录 | 新目录 | 说明 |
| --- | --- | --- |
| `domain/entities.py` | `app_v2/domain/entities.py` | SQLAlchemy ORM 实体 |
| `infrastructure/file_storage_service.py` | `app_v2/infrastructure/file_storage_service.py` | MinIO 文件存储服务 |
| `infrastructure/vector_store_service.py` | `app_v2/infrastructure/vector_store_service.py` | Qdrant 向量库服务 |
| `infrastructure/orm_session.py` | `app_v2/infrastructure/orm_session.py` | ORM Engine 和 Session 管理 |
| `infrastructure/id_generator.py` | `app_v2/infrastructure/id_generator.py` | 业务 ID 生成 |

### 3.3 训练支持代码收口

| 旧目录 | 新目录 | 说明 |
| --- | --- | --- |
| `training/schemas.py` | `app_v2/application/training_support/schemas.py` | 销售训练请求响应模型 |
| `training/repository.py` | `app_v2/application/training_support/repository.py` | 训练资料兼容仓储 |
| `training/llm_ingest.py` | `app_v2/application/training_support/llm_ingest.py` | LLM 兜底切分 |
| `training/quality.py` | `app_v2/application/training_support/quality.py` | 入库质量评估 |
| `training/publish_validation.py` | `app_v2/application/training_support/publish_validation.py` | 发布校验 |
| `training/factories/` | `app_v2/application/training_support/factories/` | 入库策略工厂 |
| `training/strategies/` | `app_v2/application/training_support/strategies/` | 入库策略实现 |

`training_support` 的命名表示这些代码是销售训练应用层的支撑能力，而不是新的顶层业务入口。

## 4. 导入路径规则

迁移后新增约定：

```text
from app_v2.domain.entities import ...
from app_v2.infrastructure.orm_session import ...
from app_v2.application.training_support.schemas import ...
from core.rag.document_parser import ...
from core.model.factory import ...
from core.utils.logger_handler import ...
from core.agent.react_agent import ...
```

禁止新增以下顶层导入：

```text
from domain...
from infrastructure...
from training...
from rag...
from model...
from utils...
from agent...
```

## 5. 设计模式说明

- 外观模式：`api/main.py` 继续作为应用启动外观，只负责创建 FastAPI、挂载 V2 路由和执行预热。
- 适配器模式：`app_v2/infrastructure/adapters/*` 继续隔离 MinIO、Qdrant 等外部系统。
- 策略模式：`app_v2/application/training_support/strategies/*` 保留不同训练资料入库策略。
- 工厂模式：`core/model/factory.py` 和 `training_support/factories/*` 负责创建模型实例和入库策略。
