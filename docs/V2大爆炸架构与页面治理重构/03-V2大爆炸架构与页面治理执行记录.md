## 旧结构清理审计

当前不能直接删除以下旧模块，因为 V2 服务仍在委托它们承接真实业务能力：

- `api.services.chat_services`：V2 聊天服务仍复用会话准备、直答判断、流式输出和保存消息逻辑。
- `api.services.upload_services`、`api.services.indexing_services`、`api.services.document_asset_service`：V2 知识资产仍复用 MinIO 预览、入库、索引和删除资产逻辑。
- `rag.knowledge_store`：V2 字典、会话、文件、训练服务仍把它当作底层数据仓储。
- `training.services.sales_training_service`：V2 销售训练服务已拆外观入口，但资料、画像、目标、会话和评分的真实实现仍在旧服务里。
- `api.routers.exam`：V2 当前仍直接挂载旧考试路由，需要后续拆到 `app_v2.application.exam_service` 后再删。

下一轮清理建议：优先把 `KnowledgeStore` 的 SQL 拆到 V2 repository，再把 `SalesTrainingService` 的超大类按资料、画像、目标、会话、评分迁移到 V2 服务，最后删除旧兼容模块。
