# V2 大爆炸架构与页面治理重构文档目录

> 本目录用于归档 V2 架构治理、页面治理、全量 V2 化迁移和旧依赖退场相关设计。  
> 如果是具体业务页面设计，例如左侧菜单、用户管理、知识资产页面，应放在对应页面目录；如果是跨模块迁移路线、架构边界、回归标准，应放在本目录。

## 1. 文档清单

| 文件 | 内容 | 使用场景 |
| --- | --- | --- |
| `01-V2大爆炸架构与页面治理设计.md` | V2 分层、后端架构、前端页面治理和统一入口设计 | 评审整体架构时先读 |
| `02-V2大爆炸架构与页面治理实施计划.md` | 初始实施步骤、任务拆解和验证方式 | 按计划推进第一轮重构时使用 |
| `03-V2大爆炸架构与页面治理执行记录.md` | 实际执行过程、改动范围、验证命令和结果 | 每次阶段性收口后更新 |
| `04-V2重构整改复盘与尾巴清单.md` | 已完成整改、遗留尾巴、风险和后续建议 | 决定下一阶段优先级时阅读 |
| `05-全量V2化三周迁移设计.md` | 3 周全量 V2 化路线、模块顺序、阶段验收标准 | 规划 2 到 4 周迁移节奏时阅读 |
| `06-全量V2化执行检查清单.md` | 每周检查项、旧依赖扫描、后端/前端回归矩阵、删除准入 | 每周验收和最终收口时使用 |

## 2. 推荐阅读顺序

1. 先读 `01-V2大爆炸架构与页面治理设计.md`，确认整体架构和页面治理原则。
2. 再读 `04-V2重构整改复盘与尾巴清单.md`，确认当前真实遗留问题。
3. 规划迁移节奏时阅读 `05-全量V2化三周迁移设计.md`。
4. 执行每周收口时使用 `06-全量V2化执行检查清单.md`。
5. 每次实际改动后更新 `03-V2大爆炸架构与页面治理执行记录.md`。

## 3. 状态口径

| 状态 | 含义 |
| --- | --- |
| 已设计 | 文档已经给出目标结构、接口、数据表或验收规则 |
| 已迁移 | 生产代码已经走 V2 路由、V2 service、V2 repository 或 adapter |
| 冻结旧实现 | 旧文件还在，但不允许继续承接新增业务逻辑 |
| 待删除 | 已无生产调用点，等待回归确认后删除 |

后续更新文档时，必须明确使用以上状态之一，避免把计划、已完成和历史兼容描述混在一起。
