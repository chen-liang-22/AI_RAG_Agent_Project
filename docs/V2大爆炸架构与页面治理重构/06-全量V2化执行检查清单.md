# 全量 V2 化执行检查清单

> 本文档用于把 `05-全量V2化三周迁移设计.md` 转成可执行、可验收的检查项。  
> 每次收口时按本清单核对，不以“文件已经存在”作为完成标准，只以生产调用路径、测试结果和旧依赖扫描为准。

## 1. 执行范围

| 范围 | 检查点 |
| --- | --- |
| 后端入口 | 生产接口统一挂在 `/api/v2/*` |
| 应用服务 | 路由层只调用 `app_v2/application/*` |
| 仓储层 | MySQL 读写通过 `app_v2/infrastructure/repositories/*` |
| 基础设施 | MinIO、Qdrant、Redis、LLM SDK 通过 adapter 或集中服务隔离 |
| 前端页面 | 页面请求统一调用 V2 API client，不直接拼旧接口路径 |
| 文档 | 设计、执行记录、尾巴清单和实际代码状态一致 |

## 9. 删除旧代码前最终确认

删除旧文件前逐项确认：

1. `app_v2` 没有导入该旧文件。
2. 前端没有调用该旧文件暴露的旧接口。
3. OpenAPI 不再需要暴露对应旧路径。
4. 已有测试覆盖 V2 替代路径。
5. 目标测试和后端全量测试通过。
6. 删除原因和替代路径已经写入执行记录或尾巴清单。

只要其中任意一项不满足，就先冻结旧实现，不直接删除。
