# 数据库设计

本文档说明新环境初始化时需要创建的 MySQL 表，以及 MySQL、Qdrant、MinIO 在项目里的分工。

## 存储分工

| 存储 | 作用 | 保存内容 |
| --- | --- | --- |
| MySQL | 业务元数据和状态库 | 文件台账、会话记录、字典、考试记录、销售训练方案和训练记录 |
| Qdrant | 向量检索库 | 知识库切片向量、训练资料切片向量、向量 payload |
| MinIO | 文件对象存储 | 用户上传的原始文件 |
| Redis | 临时状态和缓存 | 上传预览状态、短期缓存、分布式锁 |

## 初始化文件

| 文件 | 作用 |
| --- | --- |
| `mysql初始化建表和基础数据.sql` | 根据当前 MySQL 实表结构整理，创建数据库、业务表、索引、外键，并写入系统默认字典、菜单、角色和默认用户 |
| `接口文档.md` | 当前 FastAPI 接口说明 |
| `数据库设计.md` | 当前数据库和向量库设计说明 |

## 表设计

| 表名 | 中文名 | 作用 |
| --- | --- | --- |
| `documents` | 普通知识库文件元数据表 | 记录上传文件的台账信息，包含 MinIO 位置、文件 MD5、入库状态、Qdrant collection、切片数量等 |
| `conversations` | 聊天会话表 | 保存普通 RAG 聊天会话的基础信息和状态 |
| `conversation_messages` | 聊天消息表 | 保存普通 RAG 聊天中的用户消息、助手消息和消息顺序 |
| `dictionary_items` | 系统字典表 | 保存前端下拉选项、枚举配置、画像字典、上传类型等系统字典项 |
| `system_users` | 系统登录用户表 | 保存系统用户账号、密码哈希、角色和状态 |
| `system_roles` | 系统角色表 | 保存内置角色和角色状态，例如管理员、普通用户 |
| `system_menus` | 系统菜单表 | 保存左侧菜单、页面键、图标、权限编码和展示顺序 |
| `system_role_menus` | 系统角色菜单关系表 | 保存角色和菜单的授权关系 |
| `exam_sessions` | 知识掌握度测评会话表 | 保存一次知识测评的会话配置、来源文件、轮次、分数和状态 |
| `exam_questions` | 知识掌握度测评题目表 | 保存测评题目、用户答案、正确答案、得分和判题分析 |
| `training_knowledge_batches` | 销售训练资料上传批次表 | 保存销售训练资料上传批次、版本、发布状态、关联文件和切片统计 |
| `training_plans` | 销售训练方案表 | 保存销售训练方案、学员画像、客户画像选择、场景描述和生成状态 |
| `training_role_profiles` | 销售训练 AI 客户角色表 | 保存生成后的 AI 客户角色画像、可见画像、隐藏画像和证据 |
| `training_goal_settings` | 销售训练阶段和评分设置表 | 保存训练阶段、训练目标、轮次限制和评分规则 |
| `sales_training_sessions` | 销售训练会话表 | 保存一次销售陪练会话的状态、当前阶段、总分和复盘报告 |
| `sales_training_turns` | 销售训练对话轮次表 | 保存销售陪练每一轮对话、检索证据、阶段判断和教练分析 |
| `sales_training_scores` | 销售训练评分表 | 保存销售陪练最终评分、等级、是否通过和评分明细 |

## 向量库设计

Qdrant 不通过 SQL 初始化。项目启动后会按配置连接 Qdrant，上传文件时再写入向量数据。

| Collection | 作用 |
| --- | --- |
| `agent` | 默认知识库 collection，保存普通知识问答切片 |
| 训练资料 collection | 保存销售训练资料切片，具体名称以配置和上传逻辑为准 |

Qdrant point 通常包含：

| 字段 | 说明 |
| --- | --- |
| `id` | 向量点唯一编号 |
| `vector` | embedding 后的向量 |
| `payload.document_id` | 关联 MySQL `documents.document_id` |
| `payload.filename` | 原始文件名 |
| `payload.content` | 切片文本 |
| `payload.chunk_index` | 切片序号 |
| `payload.asset_type` | 资料类型，例如普通知识或销售训练资料 |
| `payload.metadata` | 页码、标题路径、来源类型等扩展信息 |

## 文件删除链路

删除文件时，需要按同一业务对象做全链路删除：

1. 删除 MySQL 中的文件台账和关联业务记录。
2. 删除 Qdrant 中对应 `document_id` 的向量点。
3. 删除 MinIO 中对应的原始文件对象。
4. 返回删除结果，日志使用中文记录每一步状态。

## 初始化顺序

1. 启动 MySQL、Redis、MinIO、Qdrant。
2. 执行 `mysql初始化建表和基础数据.sql`。
3. 配置 `.env` 和 `config/*.yml`。
4. 启动 FastAPI。
5. 上传知识库或销售训练资料，由系统写入 MinIO、MySQL 和 Qdrant。

| `training_knowledge_batches` | 销售训练资料上传批次 |
| `training_plans`、`training_role_profiles`、`training_goal_settings` | 销售训练方案、AI 客户角色、阶段和评分设置 |
| `sales_training_sessions`、`sales_training_turns`、`sales_training_scores` | 销售陪练会话、对话轮次、评分结果 |
