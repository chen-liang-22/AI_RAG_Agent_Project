# 前端页面设计

## 1. 当前前端结构

当前左侧菜单是静态配置：

```text
src/app/navigation.ts
src/app/AppShell.vue
src/App.vue
```

当前页面切换不是 Vue Router，而是：

```ts
const activePage = ref<MainPage>('home')
```

因此第一期不建议引入路由。菜单接口返回 `page_key`，前端继续用 `activePage` 切页面。

ID 处理规则：

- 后端新增表主键使用雪花算法 ID，数据库类型是 `BIGINT`。
- 前端所有 `role_id`、`menu_id`、`parent_menu_id`、`checked_menu_ids` 统一按 `string` 处理。
- 前端不把雪花 ID 转成 `number`，避免超过 JavaScript 安全整数范围后精度丢失。

## 3. 新增 API

`src/shared/api/system.ts`：

```ts
import { request } from './http'
import type {
  SystemMenuResponse,
  SystemUserCreatePayload,
  SystemUserListResponse,
  SystemUserResponse,
  SystemUserUpdatePayload,
  SystemRoleListResponse,
  SystemRoleMenuResponse,
  SystemRoleOptionResponse,
  SystemRolePayload,
} from './types'

export function fetchCurrentUserMenus() {
  return request<SystemMenuResponse[]>('/system/menus/me')
}

export function fetchSystemUsers(params: {
  page: number
  pageSize: number
  keyword?: string
  role?: string
  status?: string
}) {
  const query = new URLSearchParams()
  query.set('page', String(params.page))
  query.set('page_size', String(params.pageSize))
  if (params.keyword) query.set('keyword', params.keyword)
  if (params.role) query.set('role', params.role)
  if (params.status) query.set('status', params.status)
  return request<SystemUserListResponse>(`/system/users?${query.toString()}`)
}

export function fetchSystemRoles(params: {
  page: number
  pageSize: number
  keyword?: string
  status?: string
}) {
  const query = new URLSearchParams()
  query.set('page', String(params.page))
  query.set('page_size', String(params.pageSize))
  if (params.keyword) query.set('keyword', params.keyword)
  if (params.status) query.set('status', params.status)
  return request<SystemRoleListResponse>(`/system/roles?${query.toString()}`)
}

export function fetchSystemRoleOptions() {
  return request<SystemRoleOptionResponse[]>('/system/roles/options')
}

export function createSystemRole(payload: SystemRolePayload) {
  return request<SystemRoleResponse>('/system/roles', {
    method: 'POST',
    body: JSON.stringify(payload),
  })
}

export function updateSystemRole(roleId: string, payload: SystemRolePayload) {
  return request<SystemRoleResponse>(`/system/roles/${roleId}`, {
    method: 'PUT',
    body: JSON.stringify(payload),
  })
}

export function fetchSystemRoleMenus(roleId: string) {
  return request<SystemRoleMenuResponse>(`/system/roles/${roleId}/menus`)
}

export function updateSystemRoleMenus(roleId: string, menuIds: string[]) {
  return request<SystemRoleMenuResponse>(`/system/roles/${roleId}/menus`, {
    method: 'PUT',
    body: JSON.stringify({ menu_ids: menuIds }),
  })
}

export function disableSystemRole(roleId: string) {
  return request<{ status: string; role_id: string }>(`/system/roles/${roleId}`, {
    method: 'DELETE',
  })
}

export function enableSystemRole(roleId: string) {
  return request<{ status: string; role_id: string }>(`/system/roles/${roleId}/enable`, {
    method: 'POST',
  })
}

export function createSystemUser(payload: SystemUserCreatePayload) {
  return request<SystemUserResponse>('/system/users', {
    method: 'POST',
    body: JSON.stringify(payload),
  })
}

export function updateSystemUser(userId: string, payload: SystemUserUpdatePayload) {
  return request<SystemUserResponse>(`/system/users/${userId}`, {
    method: 'PUT',
    body: JSON.stringify(payload),
  })
}

export function disableSystemUser(userId: string) {
  return request<{ status: string; user_id: string }>(`/system/users/${userId}`, {
    method: 'DELETE',
  })
}

export function enableSystemUser(userId: string) {
  return request<{ status: string; user_id: string }>(`/system/users/${userId}/enable`, {
    method: 'POST',
  })
}

export function resetSystemUserPassword(userId: string, password: string) {
  return request<{ status: string; user_id: string }>(`/system/users/${userId}/password`, {
    method: 'POST',
    body: JSON.stringify({ password }),
  })
}
```

### 4.2 AppShell 入参调整

`AppShell.vue` 新增：

```ts
menus: SystemMenuResponse[]
menuLoading: boolean
```

菜单渲染规则：

- `directory` 渲染为菜单分组。
- `page` 渲染为可点击菜单项。
- 如果菜单接口失败，使用 `fallbackPortalPages`。
- 未知 `page_key` 不渲染，避免点击后页面空白。

菜单样式建议：

```text
一级菜单：图标 + 名称
目录菜单：图标 + 名称 + 展开箭头
二级菜单：左侧缩进，图标尺寸稍小
当前激活：背景高亮 + 左侧细线
```

## 5. App.vue 页面接入

新增异步页面：

```ts
const UserManagementPage = defineAsyncComponent(() => import('./features/system/UserManagementPage.vue'))
const RoleManagementPage = defineAsyncComponent(() => import('./features/system/RoleManagementPage.vue'))
```

登录成功或刷新登录成功后，加载菜单：

```ts
const menus = ref<SystemMenuResponse[]>([])
const menuLoading = ref(false)

async function refreshMenus() {
  menuLoading.value = true
  try {
    menus.value = await fetchCurrentUserMenus()
  } finally {
    menuLoading.value = false
  }
}
```

页面渲染增加：

```vue
<UserManagementPage

  :current-user="currentUser"
/>

  v-else-if="activePage === 'roleManagement'"
  :theme-mode="themeMode"
  :current-user="currentUser"
/>
```

```text
页面标题：用户管理
说明区：管理系统登录账号、角色和启用状态

工具栏：

  登录账号
  展示名称
  角色
  状态
  最后登录时间
  更新时间
  操作

  编辑
  重置密码
  禁用 / 启用
```

### 表单字段

新增用户：

| 字段 | 控件 | 校验 |
| --- | --- | --- |
| 登录账号 | input | 必填，1-128 |
| 展示名称 | input | 必填，1-128 |
| 初始密码 | password input | 必填，6-128 |
| 角色 | select | 来自 `/system/roles/options` |
| 状态 | select | active/disabled |

编辑用户：

| 字段 | 控件 | 校验 |
| --- | --- | --- |
| 展示名称 | input | 必填，1-128 |
| 新密码 | password input | 可选，填了则 6-128 |
| 角色 | select | 来自 `/system/roles/options` |
| 状态 | select | active/disabled |

### 交互规则

1. 当前登录用户行不展示“禁用”按钮。
2. 保存成功后刷新列表。
3. 创建成功后清空表单并关闭弹窗。
4. 禁用用户前弹确认框。
5. 重置密码单独弹窗，避免误改用户基础信息。
6. API 返回 403 时提示“无权限访问系统管理”。
7. 页面初始化时拉取启用角色选项，用于角色筛选和表单下拉。

新增文件：

```text
src/features/system/RoleManagementPage.vue
```

### 页面布局

```text
页面标题：角色管理
说明区：维护角色基础信息和左侧菜单权限

左侧：
  [关键词输入] [状态筛选] [新增角色]
  角色列表

右侧：
  角色基础信息表单
  菜单权限树
  [保存角色] [保存菜单权限]
```

### 角色表单字段

| 字段 | 控件 | 校验 |
| --- | --- | --- |
| 角色编码 | input | 新增必填，1-64，编辑时禁用 |
| 角色名称 | input | 必填，1-128 |
| 状态 | select | active/disabled |
| 排序号 | number input | 整数 |
| 角色说明 | textarea | 最多 255 |

### 菜单授权

菜单授权使用树形勾选：

- 数据来源：`GET /api/v2/system/roles/{role_id}/menus`
- 左侧展示 `menu_tree`
- 已勾选项使用 `checked_menu_ids`
- 保存时调用 `PUT /api/v2/system/roles/{role_id}/menus`

### 交互规则

1. 内置角色展示 `内置` 标识。
2. 内置角色不展示删除按钮，只允许编辑名称、说明和菜单权限。
3. 禁用角色前弹确认框。
4. 禁用失败时展示后端原因，例如“该角色仍有启用用户，不能禁用”。
5. `admin` 角色菜单权限保存时，前端不允许取消“系统管理 / 角色管理”。
