# 接口设计

## 1. 通用约定

后端统一挂载在 V2：

```text
/api/v2
```

ID 传输规则：

- 本次新增表主键使用雪花算法 ID，数据库类型为 `BIGINT`。
- API 响应中的雪花 ID 统一转成字符串，例如 `"2033000000000010001"`。
- API 请求中的 `role_id`、`menu_ids` 也按字符串接收，后端服务层再转换为数据库 `BIGINT`。
- `system_users` 是历史表，本期不强制迁移历史主键。

新增路由建议：

```text
app_v2/api/routes/system.py
```

并在 `app_v2/api/router.py` 中注册：

```python
from app_v2.api.routes import system

router.include_router(system.router)
```

路由前缀：

```python
router = APIRouter(prefix="/system", tags=["V2 系统管理"])
```

```text
GET /api/v2/system/roles?page=1&page_size=10&keyword=admin&status=active
```

```json
      "role_code": "admin",
      "role_name": "管理员",
      "status": "active",

      "built_in": true,
      "description": "系统内置管理员角色",
      "created_at": "2026-06-27 10:00:00",
      "updated_at": "2026-06-27 10:00:00"

  "page": 1,
  "page_size": 10
}
```

### 3.2 启用角色选项

```text
GET /api/v2/system/roles/options
```

权限：

- `admin`

用途：

- 用户新增/编辑时选择角色。

响应：

```json
[
  {
    "role_code": "admin",
    "role_name": "管理员"
  },
  {
    "role_code": "user",
    "role_name": "普通用户"
  }
]
```

### 3.3 创建角色

```text
POST /api/v2/system/roles
```

权限：

- `admin`

请求：

```json
{
  "role_code": "trainer_admin",
  "role_name": "训练管理员",
  "status": "active",
  "sort_order": 30,
  "description": "负责销售陪练资料和训练方案维护",
  "menu_ids": ["2033000000000010001", "2033000000000010003"]
}
```

说明：

- `role_code` 创建后不建议修改。
- `menu_ids` 可选，传了则创建后同步角色菜单权限。

### 3.4 修改角色

```text
PUT /api/v2/system/roles/{role_id}
```

权限：

- `admin`

### 3.6 保存角色菜单权限

```text
PUT /api/v2/system/roles/{role_id}/menus
```

权限：

- `admin`

请求：

```json
{
  "menu_ids": [
    "2033000000000010001",
    "2033000000000010002",
    "2033000000000010005",
    "2033000000000010006"
  ]
}
```

说明：

- 后端整体覆盖 `system_role_menus` 中该角色的菜单关系。
- 保存时校验菜单 ID 必须存在。
- 不建议允许移除 `admin` 角色的“角色管理”菜单，避免管理员无法恢复权限。

### 3.7 禁用角色

```text
DELETE /api/v2/system/roles/{role_id}
```

权限：

- `admin`

行为：

- 不物理删除。
- 更新 `system_roles.status = disabled`。
- 如果该角色仍有启用用户，返回 400。

响应：

```json
{
  "status": "disabled",
  "role_id": "2033000000000000003"
}
```

### 3.8 启用角色

```text
POST /api/v2/system/roles/{role_id}/enable
```

权限：

- `admin`

响应：

```json
{
  "status": "active",
  "role_id": "2033000000000000003"
}
```

```json
  "page_size": 10
}
```

### 4.2 创建用户

```text
POST /api/v2/system/users
```

权限：

- `admin`

请求：

```json
{
  "username": "zhangsan",
  "display_name": "张三",
  "password": "123456",
  "role": "user",
  "status": "active"
}
```

响应：

```json
{
  "user_id": "user_xxx",
  "username": "zhangsan",
  "display_name": "张三",
  "role": "user",
  "status": "active",
  "last_login_at": null,
  "created_at": "2026-06-27 10:00:00",
  "updated_at": "2026-06-27 10:00:00"
}
```

错误：

| 场景 | 状态码 | 提示 |
| --- | --- | --- |
| 账号已存在 | 400 | 登录账号已存在 |
| 角色不存在或已停用 | 400 | 角色不存在或已停用 |
| 密码过短 | 422 | 参数校验失败 |
| 非管理员 | 403 | 无权限访问系统管理 |

### 4.3 修改用户

```text
PUT /api/v2/system/users/{user_id}
```

权限：

- `admin`

请求：

```json
{
  "display_name": "张三",
  "role": "user",
  "status": "active",
  "password": null
}
```

说明：

- `password` 不传或为空，不修改密码。
- `role` 必须来自启用的 `system_roles`。
- 修改 `status` 时禁止把当前登录用户改成 `disabled`。

### 4.4 重置密码

```text
POST /api/v2/system/users/{user_id}/password
```

权限：

- `admin`

请求：

```json
{
  "password": "newPassword123"
}
```

响应：

```json
{
  "status": "ok",
  "user_id": "user_xxx"
}
```

### 4.5 禁用用户

```text
DELETE /api/v2/system/users/{user_id}
```

权限：

- `admin`

行为：

- 不物理删除。
- 更新 `system_users.status = disabled`。

响应：

```json
{
  "status": "disabled",
  "user_id": "user_xxx"
}
```

错误：

| 场景 | 状态码 | 提示 |
| --- | --- | --- |
| 用户不存在 | 404 | 用户不存在 |
| 禁用当前登录用户 | 400 | 不能禁用当前登录用户 |
| 非管理员 | 403 | 无权限访问系统管理 |

### 4.6 启用用户

```text

```

权限：

- `admin`

响应：


## 5. Pydantic 模型设计

建议新增到 `app_v2/domain/schemas.py`，如果文件继续变大，再拆为 `app_v2/domain/system_schemas.py`。

```python
class SystemMenuResponse(BaseModel):
    """系统菜单响应，雪花 ID 按字符串返回给前端。"""

    menu_id: str
    parent_menu_id: str | None = None
    menu_code: str
    menu_name: str
    menu_type: str
    page_key: str | None = None
    route_path: str | None = None
    component_key: str | None = None
    icon: str | None = None
    permission_code: str | None = None
    sort_order: int
    visible: bool
    status: str
    metadata: dict = Field(default_factory=dict)
    children: list["SystemMenuResponse"] = Field(default_factory=list)
```

```python
class SystemUserResponse(BaseModel):
```

## 6. 后端服务设计

### SystemRepository

职责：

- 查询角色可见菜单。
- 查询全部菜单。
- 查询角色分页。
- 查询启用角色选项。
- 按角色编码查重。
- 创建角色。
- 修改角色基础信息。
- 修改角色状态。
- 查询角色已授权菜单。
- 覆盖保存角色菜单关系。
- 查询用户分页。
- 按账号查重。
- 创建用户。
- 修改用户基础信息。
- 修改密码。
- 修改用户状态。

### SystemApplicationService

职责：

- 组装菜单树。
- 校验角色状态枚举。
- 校验用户角色是否存在且启用。
- 校验角色是否仍有关联启用用户。
- 校验管理员权限。
- 拦截禁用当前用户。
- 拦截删除内置角色。
- 拦截移除管理员角色关键菜单。
- 调用 `create_password_hash` 生成密码哈希。

### 权限辅助函数

可以先放在 `app_v2/api/routes/system.py`：

```python
def require_admin(user: SystemUserEntity) -> None:
    """校验当前用户是否为管理员。"""

    if user.role != "admin":
        raise HTTPException(status_code=403, detail="无权限访问系统管理")
```

后续权限点增多后，再提取到 `app_v2/api/dependencies.py`。
