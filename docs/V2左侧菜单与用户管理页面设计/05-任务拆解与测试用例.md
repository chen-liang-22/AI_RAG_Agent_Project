## 2. 后端测试用例

### 菜单接口

| 编号 | 用例 | 预期 |
| --- | --- | --- |
| M-01 | admin 查询 `/system/menus/me` | 返回全部菜单，包括系统管理、用户管理、角色管理、菜单管理 |
| M-02 | user 查询 `/system/menus/me` | 不返回系统管理、用户管理、角色管理、菜单管理 |
| M-03 | 未登录查询菜单 | 返回 401 |
| M-04 | 菜单设置 `visible=0` | 当前用户菜单中不返回该菜单 |
| M-05 | 菜单设置 `status=disabled` | 当前用户菜单中不返回该菜单 |
| M-06 | 二级菜单排序 | 子菜单按 `sort_order` 升序 |
| M-07 | 创建菜单 | 成功写入 `system_menus`，接口返回字符串雪花 ID |
| M-08 | 修改菜单 | 成功更新名称、排序、显示状态和扩展配置 |
| M-09 | 禁用仍有子菜单的菜单 | 返回 400 |
| M-10 | 禁用叶子菜单 | 菜单状态变成 `disabled` |
| M-11 | 启用菜单 | 菜单状态变成 `active` |
| M-12 | 删除菜单 | 叶子菜单被物理删除并清理角色授权 |
| M-13 | 删除核心系统菜单 | 返回 400 |

## 4. 手动验收步骤

1. DONE：执行菜单和角色迁移 SQL。
2. 启动后端。
3. 使用默认管理员登录：`admin / 1234qwer`。
4. 确认左侧菜单出现“系统管理 / 用户管理 / 角色管理”。
5. 在角色管理中创建一个测试角色，并勾选首页和智能客服。
6. 在用户管理中创建一个使用测试角色的用户。
7. 退出管理员，使用测试用户登录。
8. 确认测试用户只看到已授权菜单。
9. 管理员重新登录，禁用测试用户。
10. 使用被禁用用户登录，确认登录失败。
11. 管理员启用测试用户，确认测试用户可重新登录。
