# V2 大爆炸重构实施计划

## 目标

把当前项目从历史迭代堆叠结构重构为 V2 分层架构：后端接口层、应用服务层、仓储层、基础设施适配层边界清晰；前端按首页、智能客服、知识资产、销售训练、字典、考试拆分为模块。旧兼容层直接删除，字典内容保留。

## 全局约束

- 日志统一中文输出。
- API 字段名、数据库字段名、SSE 事件名仍使用英文协议名。
- MinIO 是唯一文件持久化处理器。
- 字典内容原样保留。
- 业务测试数据可以重建。
- 接口允许破坏式改造，前端同步使用 V2 client。
- 深色和浅色主题必须同时可读。
- Python 代码给初学者友好的关键注释，但不写翻译式废话。

## 风险控制

| 风险 | 控制方式 |
| --- | --- |
| 大量接口变化导致前端不可用 | 前后端同一轮切换到 V2 client |
| 旧测试不再匹配新接口 | 删除旧兼容测试，保留核心流程测试 |
| 本地数据库仍有旧业务数据 | 业务数据允许重建，字典内容保留 |
| Qdrant 旧 collection 混乱 | 知识资产页面提供重建索引入口 |
| 前端页面再次变大 | 单个 Vue 文件超过 800 行就继续拆组件 |
