# 异步入库实施计划

## 目标

放弃 XXL-JOB，使用项目内部 `ingest_tasks` 任务台账和 FastAPI 后台线程处理文件入库。

## 实施范围

1. 新增 `ingest_tasks` ORM 实体、仓储、服务和查询接口。
2. 知识库确认入库创建任务并快速返回。
3. 销售训练上传创建任务并快速返回，后台生成临时切片和质量评分。
4. 销售训练重新切分改为创建任务。
5. 删除 XXL-JOB 运行依赖和文档残留。
6. 补充单元测试和编译验证。

## 设计模式

1. 外观模式：上传服务仍然作为路由入口，隐藏 MySQL、MinIO、Qdrant 和后台任务细节。
2. 策略模式：任务服务按 `task_type` 分派普通知识库入库、销售训练入库、销售训练重切。
3. 状态模式：`queued/running/succeeded/failed` 和 `current_step` 共同决定页面按钮可用性。
4. 模板方法模式：任务执行统一经历领取、运行、进度更新、成功/失败落库。
