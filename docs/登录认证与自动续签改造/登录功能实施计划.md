# 登录功能实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现一套完整登录能力：JWT access token、HttpOnly refresh token、Redis refresh session、自动续签、退出吊销。

**Architecture:** 后端用 `AuthService` 做认证外观，`AuthRepository` 管 MySQL 用户，`RefreshSessionStore` 管 Redis 会话。前端由 `App.vue` 做登录门禁，`api.ts` 统一处理 access token、401 自动 refresh 和请求重试。

**Tech Stack:** FastAPI, SQLAlchemy ORM, Redis, PyJWT, PBKDF2-HMAC-SHA256, Vue 3, TypeScript, Element Plus.


保存格式：


- [ ] **Step 4: 实现 JWT access token**

使用 `jwt.encode` / `jwt.decode`：

```text
algorithm = HS256
secret = AUTH_TOKEN_SECRET
exp = 当前时间 + AUTH_ACCESS_TOKEN_EXPIRE_SECONDS
type = access
sub = user_id
```

- [ ] **Step 5: 实现 refresh token**

```text
create_refresh_token() -> secrets.token_urlsafe(48)
hash_refresh_token(token) -> sha256 hex
Redis key -> auth:refresh:{hash}
TTL -> AUTH_REFRESH_TOKEN_EXPIRE_SECONDS
```

- Modify: `api/main.py`

- [ ] **Step 1: 实现 `/auth/login`**

流程：

```text
1. 校验用户名密码
2. 签发 access_token
3. 创建 refresh_token
4. Redis 保存 refresh session
5. Set-Cookie 写入 refresh_token
6. 返回 access_token 和 user
```

- [ ] **Step 2: 实现 `/auth/refresh`**

流程：

```text
1. 从 Cookie 读取 refresh_token
2. Redis 校验 refresh session
3. 查询用户状态
4. 返回新的 access_token
```

- [ ] **Step 3: 实现 `/auth/me`**

通过 `get_current_user` 校验 Bearer JWT，并返回当前用户。

- [ ] **Step 4: 实现 `/auth/logout`**

流程：

```text
1. 从 Cookie 读取 refresh_token
2. 删除 Redis refresh session
3. 清除 Cookie
4. 返回 {"status": "ok"}
```

- [ ] **Step 5: 注册路由**

`api/main.py`：
