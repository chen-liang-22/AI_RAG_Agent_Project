# 接口文档

本文档按业务大类整理当前 FastAPI 项目暴露的接口、请求字段、响应字段、字段中文含义和主要枚举值。

## 1. 基础说明

| 项目 | 说明 |
| --- | --- |
| 服务框架 | FastAPI |
| 应用入口 | `api/main.py` |
| OpenAPI 标题 | `知习台 API` |
| 当前版本 | `1.0.0` |
| 默认本地地址 | `http://127.0.0.1:8000` 或 `http://localhost:8000` |
| 路由前缀 | 当前后端代码没有统一 `/api` 前缀，本文档按真实后端路径书写 |
| 普通响应格式 | JSON |
| 流式响应格式 | SSE 文本流，`Content-Type: text/event-stream` |

当前路由模块：

| 路由模块 | 大类 | 主要职责 |
| --- | --- | --- |
| `api/routers/health.py` | 健康检查 | 检查后端和 Qdrant 状态 |
| `api/routers/chat.py` | 聊天和会话 | 一次性聊天、流式聊天、聊天记录、检索调试 |
| `api/routers/knowledge.py` | 知识库上传和管理 | 上传预览、确认入库、文件列表、预览、删除、重建索引 |
| `api/routers/dictionaries.py` | 字典配置 | 查询和维护系统枚举 |
| `api/routers/exam.py` | 考试 | 题源目录、开始考试、提交答案、考试历史 |
| FastAPI 内置路由 | 系统文档 | OpenAPI JSON、Swagger UI、ReDoc 文档页面 |

## 2. 全局枚举

### 2.1 服务状态 `service_status`

| 枚举值 | 中文含义 | 使用位置 |
| --- | --- | --- |
| `ok` | 正常 | `/health` 的 `status`、`qdrant` |
| `degraded` | 降级 | `/health` 的 `status`，表示部分依赖不可用 |
| `unavailable` | 不可用 | `/health` 的 `qdrant` |

### 2.2 模型档位 `model_mode`

| 枚举值 | 中文含义 | 当前配置模型 | 说明 |
| --- | --- | --- | --- |
| `high` | 高质量 | `deepseek-v4-flash` | 更看重回答质量，首字可能更慢 |
| `medium` | 均衡 | `qwen3.6-flash` | 平衡质量和速度 |
| `low` | 低延迟 | `qwen-turbo` | 更看重响应速度，也用于部分推荐场景 |

说明：

1. 前端不传 `model_mode` 时，后端会从 `model_mode` 字典里取默认项。
2. 当前字典默认项是 `high`。
3. 当前推荐项是 `low`，上传文件模型推荐可能使用该档位。

### 2.3 聊天路由模式 `chat_route_mode`

该值来自 `config/rag.yml`，不是接口字段，但会影响 `/chat` 和 `/chat/stream` 的实际执行链路。

| 枚举值 | 中文含义 | 说明 |
| --- | --- | --- |
| `direct_rag` | 知识库直答 | 用户问题直接走“Qdrant 检索 -> 精排 -> 最终模型回答” |
| `agent` | Agent 工具链 | 走 LangGraph/ReAct Agent，由模型判断是否调用工具 |
| `react_agent` | Agent 工具链别名 | 等价于 `agent` |
| `legacy_agent` | Agent 工具链旧别名 | 等价于 `agent` |

### 2.4 文档结构类型 `document_type`

| 枚举值 | 中文含义 | 说明 |
| --- | --- | --- |
| `text` | 普通文本型 | 没有稳定问答结构的普通文档 |
| `qa` | 问答型 | 按问题和答案组织的文档，例如 FAQ、100 问 |
| `numbered` | 编号条目型 | 按编号条目组织，但不一定是严格问答 |

### 2.5 分片策略 `split_strategy`

| 枚举值 | 中文含义 | 说明 |
| --- | --- | --- |
| `recursive` | 递归通用切分 | 按段落、换行、标点和长度递归切分 |
| `numbered_qa` | 编号问答切分 | 把“第 N 问/答案”这类结构切成 QA 分片 |
| `outline_qa` | 目录问答切分 | 根据 PDF 书签目录或章节结构生成 QA 分片 |
| `numbered_segments` | 编号条目切分 | 把编号条目切成普通知识分片 |

### 2.6 知识库文件状态 `document_status`

| 枚举值 | 中文含义 | 说明 |
| --- | --- | --- |
| `uploaded` | 已上传 | 文件已保存，但还没有完成 Qdrant 入库 |
| `indexing` | 入库中 | 正在解析、分片、向量化、写入 Qdrant |
| `indexed` | 已索引 | 文件已成功写入 Qdrant |
| `failed` | 入库失败 | 解析、向量化或写入 Qdrant 失败 |
| `deleted` | 已删除 | 文件被标记删除，Qdrant points 已删除或尝试删除 |

### 2.7 知识库操作结果 `knowledge_result_status`

| 枚举值 | 中文含义 | 说明 |
| --- | --- | --- |
| `indexed` | 已索引 | 单文件入库或重建成功 |
| `duplicate` | 重复 | 存在相同 MD5 的 active 文件 |
| `failed` | 失败 | 单项操作失败 |
| `ok` | 成功 | 批量操作全部成功 |
| `partial_failed` | 部分失败 | 批量操作中有部分文件失败 |

### 2.8 预览类型 `preview_type`

| 枚举值 | 中文含义 | 说明 |
| --- | --- | --- |
| `text` | TXT 文本 | 直接读取 TXT 文本内容 |
| `pdf_text` | PDF 提取文本 | 从 PDF 中提取文本后返回 |
| `unsupported` | 不支持 | 当前文件类型不支持预览 |

### 2.9 会话状态 `conversation_status`

| 枚举值 | 中文含义 | 说明 |

## 3. 健康检查接口

### 3.1 查询服务健康状态

| 项目 | 内容 |
| --- | --- |
| 方法 | `GET` |
| 路径 | `/health` |
| 大类 | 健康检查 |
| 作用 | 检查后端服务和 Qdrant 是否可用，并返回当前 collection 信息 |

请求参数：无。

响应字段：

| 字段名 | 类型 | 中文含义 | 枚举/说明 |
| --- | --- | --- | --- |
| `status` | `string` | 整体服务状态 | `ok`、`degraded` |
| `qdrant` | `string` | Qdrant 状态 | `ok`、`unavailable` |
| `collection_name` | `string` | 当前配置使用的 Qdrant collection 名称 | 默认 `agent` |
| `collections` | `string[]` | Qdrant 中已有的 collection 列表 | Qdrant 不可用时为空数组 |

响应示例：

```json
{
  "status": "ok",
  "qdrant": "ok",
  "collection_name": "agent",
  "collections": ["agent"]
}
```

## 4. 聊天接口

### 4.1 一次性聊天

| 项目 | 内容 |
| --- | --- |
| 方法 | `POST` |
| 路径 | `/chat` |
| 大类 | 聊天 |
| 作用 | 用户发起一次聊天请求，后端生成完整回答后一次性返回 |

请求体 `ChatRequest`：

| 字段名 | 类型 | 是否必填 | 中文含义 | 枚举/限制 |
| --- | --- | --- | --- | --- |
| `message` | `string` | 是 | 用户本轮输入的问题 | 最小长度 1 |
| `user_id` | `string \| null` | 否 | 用户编号 | 可为空，后续可接入登录体系 |
| `conversation_id` | `string \| null` | 否 | 当前会话编号 | 为空时后端创建新会话 |
| `model_mode` | `string \| null` | 否 | 回答模型档位 | `high`、`medium`、`low`，为空用默认档位 |
| `collection_name` | `string \| null` | 否 | 本次检索使用的 Qdrant collection | 为空使用默认 collection |

请求示例：

```json
{
  "message": "扫地机器人适合有宠物的家庭吗？",
  "user_id": "user_001",
  "conversation_id": null,
  "model_mode": "high",
  "collection_name": "agent"
}
```

响应字段 `ChatResponse`：

| 字段名 | 类型 | 中文含义 | 说明 |
| --- | --- | --- | --- |
| `answer` | `string` | 完整回答 | 后端最终生成的自然语言回答 |
| `conversation_id` | `string` | 当前会话编号 | 前端继续聊天需要继续携带 |
| `first_token_ms` | `number \| null` | 首字耗时 | 一次性接口没有真实首字，当前通常等于总耗时 |
| `total_ms` | `number \| null` | 总耗时 | 从请求开始到完整回答结束的耗时，单位毫秒 |

响应示例：

```json
{
  "answer": "可以。宠物家庭建议优先选择大吸力、防毛发缠绕、尘盒容量较大的扫地机器人。",
  "conversation_id": "conv_xxx",
  "first_token_ms": 2350.42,
  "total_ms": 2350.42
}
```

### 4.2 流式聊天

| 项目 | 内容 |
| --- | --- |
| 方法 | `POST` |
| 路径 | `/chat/stream` |
| 大类 | 聊天 |
| 作用 | 用户发起聊天请求，后端通过 SSE 持续返回 token/chunk |
| 响应类型 | `text/event-stream` |

请求体：同 `/chat` 的 `ChatRequest`。

响应头：

| 响应头 | 中文含义 |
| --- | --- |
| `Cache-Control: no-cache, no-transform` | 禁止缓存和代理转换，降低流式内容被缓冲的概率 |
| `Connection: keep-alive` | 保持连接不断开 |
| `X-Accel-Buffering: no` | 部署到 Nginx 时关闭响应缓冲 |

SSE 事件：

| 事件名 | data 字段 | 中文含义 |
| --- | --- | --- |
| `meta` | `conversation_id` | 当前会话编号 |
| `meta` | `mode` | 流式链路模式，知识库直答时为 `direct_rag`；Agent 流式可能不返回该字段 |
| `meta` | `collection_name` | 当前检索的 Qdrant collection，知识库直答链路返回 |
| `meta` | `model_mode` | 当前模型档位 |
| `meta` | `model_name` | 当前模型名称 |
| `metric` | `first_token_ms` | 首个内容片段返回耗时，单位毫秒 |
| `chunk` | `content` | 本次增量文本片段 |
| `done` | `done` | 是否完成，完成时为 `true` |
| `done` | `conversation_id` | 当前会话编号 |
| `done` | `collection_name` | 当前 collection，知识库直答链路返回 |
| `done` | `model_mode` | 当前模型档位 |
| `done` | `model_name` | 当前模型名称 |
| `done` | `first_token_ms` | 首字耗时 |
| `done` | `total_ms` | 总耗时 |
| `error` | `error` | 错误信息 |

SSE 示例：

```text
event: meta
data: {"conversation_id":"conv_xxx","mode":"direct_rag","collection_name":"agent","model_mode":"high","model_name":"deepseek-v4-flash"}

event: metric
data: {"first_token_ms":1200.5}

event: chunk
data: {"content":"可以。"}

event: done
data: {"done":true,"conversation_id":"conv_xxx","collection_name":"agent","model_mode":"high","model_name":"deepseek-v4-flash","first_token_ms":1200.5,"total_ms":3600.9}
```

### 4.6 RAG 检索调试

| 项目 | 内容 |
| --- | --- |
| 方法 | `POST` |
| 路径 | `/debug/retrieve` |
| 大类 | 调试 |
| 作用 | 调试 RAG 检索链路，只返回检索和精排信息，不生成最终回答 |

请求体 `DebugRetrieveRequest`：

| 字段名 | 类型 | 是否必填 | 中文含义 | 限制 |
| --- | --- | --- | --- | --- |
| `query` | `string` | 是 | 要调试的用户问题 | 最小长度 1 |

响应字段：  
该接口直接返回 `RagSummarizeService.debug_retrieve()` 的字典结果，当前不是固定 Pydantic 模型。常见字段包括意图分析、query 改写、metadata filter、候选资料、向量分数、精排分数等。

注意：  
该接口会暴露检索细节，生产环境建议加鉴权或关闭。

## 5. 知识库上传和管理接口

### 5.1 上传文件预览

| 方法 | `POST` |
| 路径 | `/knowledge/upload/preview` |
| 大类 | 知识库上传 |
| 作用 | 上传文件到临时目录，识别文档类型和推荐切分策略，等待用户确认后再正式入库 |
| 请求类型 | `multipart/form-data` |

Form 参数：

| 参数名 | 类型 | 是否必填 | 中文含义 | 限制 |

| `file` | `UploadFile` | 是 | 上传文件 | 当前允许 `txt`、`pdf` |

| 字段名 | 类型 | 中文含义 | 枚举/说明 |

### 5.2 模型推荐文档类型和切分策略
